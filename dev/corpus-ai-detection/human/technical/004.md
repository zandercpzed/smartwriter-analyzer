## Text

Implementar autenticação OAuth2 pela primeira vez é intimidador. A especificação é longa, há muitos fluxos diferentes, e é fácil se perder nos detalhes. Mas na prática, o fluxo de código de autorização (Authorization Code Flow) que você provavelmente vai usar é bem mais simples do que parece.

Vou começar pelo conceito: você quer que seus usuários façam login usando sua conta Google, sem que sua aplicação nunca saiba a senha deles. Para isso, você redireciona o usuário para o Google, ele autentica lá, e o Google te diz "sim, esse é o usuário X" via um código que você pode converter em um token de acesso.

No seu servidor, você vai fazer essas requisições HTTP:

1. Quando o usuário clica "Logar com Google", você o redireciona para:
   `https://accounts.google.com/o/oauth2/v2/auth?client_id=SEU_ID&redirect_uri=...`

2. Depois que ele autentica, Google o redireciona de volta para sua URL com um código.

3. Seu servidor então faz uma requisição POST para trocar aquele código por um token:
   `https://oauth2.googleapis.com/token`

Aqui está o ponto crucial: essa requisição POST não deve vir do navegador do usuário. Ela deve vir do seu servidor backend. Por quê? Porque você precisa incluir seu client_secret, que é... bem, secreto. Nunca exponha isso ao cliente.

Implementar isso em Node.js com Express é bem direto. Use a biblioteca `passport-google-oauth20` se quiser algo pronto, ou implemente manualmente se quer entender cada passo. Pessoalmente, eu prefiro a segunda abordagem para meus projetos, porque ajuda a entender o fluxo de verdade.

Um detalhe importante: armazene o token de forma segura. Use cookies HttpOnly, não localStorage. Tokens são secrets valiosos; localStorage pode ser acessado por qualquer script na página se houver uma vulnerability XSS.
